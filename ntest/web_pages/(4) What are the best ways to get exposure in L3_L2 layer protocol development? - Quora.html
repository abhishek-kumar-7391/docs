          If you just want to get a hang of the networking concepts and not really need to dabble into production code (or don't have access to production code) then you can start off with download Mininet and POX (OpenFlow Controller).

          Here are some references:
          • http://openflow.stanford.edu/dis...
          • https://github.com/mininet/openf...

          There are many examples and code that simulate L2 and L3 capabilities such as packet filtering, routing, spanning tree and many others. There are many networking courses (including mine) that use this combination to teach networking concepts to our students.

          As always, diving in is the key step.  Just do it.

          Start by doing maintenance on it. Fix some bugs, spend time in gdb with it, and pretty soon you will grok the code. When you reach a comfort level with that, it'll be time to look at a different protocol.

          If you choose to download someone else's code, please be forewarned: the open source code that's available is not the best possible example. Working with some proprietary code will show you other techniques.
